## 问题

什么是逃逸分析？Go 编译器如何决定变量分配在栈还是堆？

## 答案

逃逸分析（Escape Analysis）是 Go 编译器在编译期进行的一项优化技术，用于决定变量应该分配在栈上还是堆上。栈分配成本极低（移动栈指针），函数返回自动回收；堆分配需要垃圾回收器介入，成本较高。

常见触发逃逸到堆的场景有五种：1）返回局部变量的指针（变量生命周期超出函数）；2）闭包捕获的变量（可能被多个 goroutine 引用）；3）接口装箱（赋给 interface{} 时类型信息需要堆分配）；4）发送到 channel 的值；5）作为 slice/map 的值且可能扩容。

可以使用 `go build -gcflags="-m"` 查看逃逸分析结果。输出中的 `moved to heap` 表示变量逃逸，`does not escape` 表示留在栈上。理解逃逸分析有助于编写高性能代码：尽量避免不必要的指针返回，使用具体类型而非 interface{}，预分配 slice 容量避免扩容。

一个经典例子：`func f() *int { x := 42; return &x }` 中 x 会逃逸到堆，因为返回了它的地址。而 `func f() int { x := 42; return x }` 中 x 留在栈上。
